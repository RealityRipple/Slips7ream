Section 1.10.4

Extraction Error

SLIPS7REAM deals with many forms of compressed files, and occasionally there are problems when dealing with compression. Most often, these problems are caused by corrupt files or unexpected compression types. SLIPS7REAM tries to notify you about any such problems with one of the error messages below.

If you receive a CRC Error, then the file has been modified somehow, and the compressed data may not be what you expect, or it may be corrupt. CRC stands for Cyclic Redundancy Check, a simple algorithm that generates a (relatively) small "checksum" number based on the contents of the file, allowing software to quickly determine if anything has changed in the file. One bit out of place will produce a drastically different result, and if the checksum values are at all different, an error is thrown.

You may want to retry the file, in case the error was a mistake, but usually this error means the compressed file in question is bad. You should re-download the file from its original source over a good network connection to ensure a clean copy.

If SLIPS7REAM throws a Data Error during extraction of a compressed file, the compressed data is probably corrupted. Basically, this error is the same as the CRC error above, except that the file you're working with doesn't have a CRC checksum, so the error cropped up during decompression instead of the initial check.

An Unsupported Method error message usually means that the file is compressed using a compression method that is foreign to SLIPS7REAM. If you receive this error message, please make sure the file in question is a standard file of its type (Windows Update, Language Pack, etc..) from Microsoft or another legitimate source. Some third-party packages may use proprietary compression methods which are not supported by SLIPS7REAM.

The File Not Found error typically means that the compressed file does not have a list of its contents, or that the file expected to be extracted does not exist in the list. Chances are in either instance that the file in question is not the right file for use in the slipstream process, such as a type of update that is not supported by SLIPS7REAM, or a third-party package.

The ISO does not have an INSTALL.WIM error is a warning that the ISO being loaded does not contain a Windows Image. Usually, this means you selected the wrong ISO by mistake. However, it could be an issue with the ISO's file system. Check the Output Console for details.

If The ISO's INSTALL.WIM file has been split up into SWM files, then SLIPS7REAM will not read it. SWM files can not be modified, so SLIPS7REAM throws this error instead of reading them.

If you receive any other error messages, they will typically include further details as to the kind of issue that occurred during extraction. Hopefully, the error message will provide you with the details necessary to resolve the problem yourself. If you receive an error you do not understand, please contact me and I'll try to assist you, and maybe include a more detailed error message in a future version of SLIPS7REAM.
