Section 1.5.3

Driver Properties

The Driver Properties Window can be a little confusing. Each driver contains some basic information about the driver and the driver's Class, as well as a list of compatible Architectures. For each architecture, there's a list of compatible Hardware, each with its own details including Hardware IDs, which contain relevant Compatible and Exclude IDs.

  • Original Name: Only included in Integrated Drivers. This was the name of the INF before it was integrated.
  • Published Name: In Integrated Drivers, this is the new name of the INF after integration. In the Updates list, this is the path where the driver INF file is located.
  • Driver Store Path: In Integrated Drivers, this is the location where the INF and any other Driver files are stored. In the Updates list, this is the same as the Published Name.
  • In-Box: Only included in Integrated Drivers. If the value is Yes, then this driver was included with the original Operating System release. If No, the driver was integrated later. In-Box drivers can not be removed.
  • Provider Name: The company that wrote the Driver. Not necessarily the same as the Hardware Manufacturer.
  • Date: This is the date the driver was written.
  • Version: This is the version of the driver. The driver version is usually the same as the OS version for drivers that came with Windows.
  • Boot Critical: If Yes, then this driver is critical for booting Windows when in use.
    • Boot-critical drivers are reflected. In other words, the files are copied into the image according to what's specified in the .inf file. The PC completes installation tasks during the initial boot, including updating the Critical Devices Database (CDDB) and the registry.
    • Drivers that aren't boot critical are staged. In other words, they're added to the driver store. After Windows starts, PnP detects the device and installs the matching driver from the driver store.
  • Driver Class
    • Name: The Class Name, identifying what kind of hardware this driver describes.
    • Description: Usually a slightly more descriptive version of the Class Name.
    • GUID: The Class GUID, which universally identifies the Driver Class.
  • Architecture: The Architecture dropdown includes all compatible CPU Architectures included in the driver. Selecting an Architecture entry will re-populate the Hardware dropdown (see Below).
  • Hardware: The Hardware dropdown includes all compatible hardware for the selected CPU Architecture included in the driver. Selecting a Hardware entry will re-populate the Hardware Description entries (see Below).
  • Hardware Description
    • Service Name: The short name of the Windows Service which this driver is attached to.
    • Architecture: Usually redundant with the Architecture dropdown (see Above).
    • Manufacturer: The name of the company that created the hardware.
    • Hardware IDs: A list of all the Hardware IDs supported by this driver. Selecting an ID will re-populate the Compatible IDs and Exclude IDs lists (see Below).
    • Compatible IDs: A list of any Hardware IDs that should be included in support along with the selected Hardware ID (see Above).
    • Exclude IDs: A list of any Hardware IDs that should be excluded from support in the range defined by the selected Hardware ID (see Above).
