Section 1.5.2

Update Properties

The Windows Update Properties Window lists details about a Windows Update:

  • Name: The name SLIPS7REAM uses in the Update List.
  • Display Name: Depends on the type of file, but is generally a common name of the update, often with a version number.
    • For MSU Files, this is the * part of the "*pkgProperties.txt" file inside the update.
    • For CAB files, this is the "displayName" value in the "update.mum" XML file.
    • For LIP files, MSI and LP.CAB Language files, this is the "identifier" value in the "update.mum" XML file.
    • EXE parsing uses either CAB or Language rules depending on what it is.
  • Identity: The Identity string contains the following data, delimited by tildes (~):
    1. Name: The Identity Name is the name of the update. It's how SLIPS7REAM tells one update from another, and how it knows to compare different versions of the same update.
    2. Token: The Token value is almost always 31bf3856ad364e35.
    3. Architecture: (Usually) redundant with the Architecture value below. If empty, then this update is "AnyCPU" or Architecture-neutral.
    4. Language: The Language this update applies to. If empty, then this update is Language-neutral.
    5. Version: (Usually) redundant with the Version value below.
  • Applies to: The Operating System or Program which the Update applies to.
  • Architecture: The OS Architecture this update is designed for. SLIPS7REAM uses this value to determine if it should be integrated into an Image Package.
  • Build Date: This is usually in the format of YEAR/MM/DD.
  • Version: The Version of most Windows Updates is 6.1.Update_Major.Update_Minor to identify them as updates for Windows 7 (or other numbers for other versions).
  • Knowledge Base: You can click the KB Article link to view details about this update, if this update has a KB Article Number, or Details if it's a Support Link.
